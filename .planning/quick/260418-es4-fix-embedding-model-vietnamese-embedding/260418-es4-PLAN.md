## Context

Hiện tại hệ thống đang chạy embedding default `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` (384-dim) ở nhiều nơi (config.py, docker-compose.yml, chat_tasks.py) trong khi yêu cầu thật là `AITeamVN/Vietnamese_Embedding_v2` (1024-dim). Đồng thời, RAG retrieval đang đọc `payload.get("content", "")` trong `hybrid_search.py` & `search_helpers.py`, NHƯNG khi upsert PDF text chunks trong `media_tasks.py` (line 345) lại lưu key `"content"` — riêng phần audio/transcript (line 456) lại dùng `"text"`. Cần đồng bộ một field name duy nhất để retrieval không trả empty.

**Bên upsert** (`media_tasks.py`):
- Line 345 (PDF chunks): `"content": text` ✅ matches retrieval
- Line 456 (audio): `"content": payload.get("text", "")` ✅ matches retrieval
- Line 191/418/564: dùng `"text"` cho parent/child/segment payload — đây là chỗ MISMATCH

→ Convention chuẩn hoá: **tất cả payload upsert dùng key `"text"`**, retrieval đọc `payload.get("text", "")`. (Theo bug doc & vì parent-child store đã dùng `text`.)

**Cách 1 — Script Python (preferred):**
- [ ] Tạo file `chatSNP170226/backend/scripts/recreate_qdrant_collections.py`:

  """One-shot script: drop & recreate Qdrant collections with new vector_size."""
  from src.core.config import get_settings
  from src.core.qdrant_setup import get_qdrant_client, ensure_collections

  settings = get_settings()
  client = get_qdrant_client()
  for name in ("chat_chunks", "port_knowledge", "vanna_schemas_openai", "mem0_memories"):
      try:
          client.delete_collection(name)
          print(f"[OK] Deleted {name}")

  ensure_collections(client, settings.embedding_dimension)
  print(f"[OK] Recreated with vector_size={settings.embedding_dimension}")
