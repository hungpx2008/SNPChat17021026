# Phase 11 Plan 01 — Docling VLM Parallelization + Converter Singleton + Stage Timing

## Goal

The VLM loop in `orchestrator.py` (lines 117–215) calls OpenRouter sequentially for each image (timeout=60s each, up to 10 images = 10 min worst case). This plan parallelizes those calls (max 4 concurrent via semaphore), adds a `DOCLING_VLM_ENABLED` env toggle to skip VLM entirely when not needed, caches the `DocumentConverter` as a module-level singleton to avoid per-task cold-starts (~10–30s), and adds structured JSON timing logs at each pipeline stage (parse / vlm / chunk / embed / upsert).

## Tasks

### Task 1: Honor `DOCLING_VLM_ENABLED=false` cleanly
**File:** `chatSNP170226/backend/src/services/docling/orchestrator.py`
**What:** The flag `vlm_enabled` is already read (line 103) but the guard at line 117 only checks `vlm_enabled and api_key and doc.pictures`. Make the toggle a top-level env default of `false` so VLM is opt-in.
**How:**
1. In `orchestrator.py`, change the `env_bool` default:
   ```python
   vlm_enabled = env_bool("DOCLING_VLM_ENABLED", False)  # default off
   ```
2. Add `.env.example` entry (and `docker-compose.yml` env section for `worker_media` service):
   ```
   DOCLING_VLM_ENABLED=false
   ```
3. When `vlm_enabled=False`, log a single `INFO` line: `[vlm] VLM disabled (DOCLING_VLM_ENABLED=false) — skipping image analysis`.
4. Ensure no image is base64-encoded / no HTTP call is made when disabled.

### Task 2: Parallelize VLM HTTP calls with `asyncio.gather` + semaphore
**File:** `chatSNP170226/backend/src/services/docling/orchestrator.py`
**What:** Replace the serial `for pic_ix, pic in enumerate(doc.pictures)` loop with a concurrent approach capped at 4 parallel calls.
**How:**
1. Extract inner loop body into an async helper `_describe_picture(pic, pic_ix, ...) -> tuple[str, int] | None`.
2. Inside `_describe_picture`, use the existing `get_http_client(timeout=60.0).post(...)` — but since `get_http_client` returns a sync `httpx.Client`, run it in a thread via `asyncio.get_event_loop().run_in_executor(None, ...)`.
3. Wrap the orchestrator's VLM section in an `asyncio.run(...)` block (the orchestrator is called from a sync Celery task):
   ```python
   import asyncio
   from asyncio import Semaphore

   async def _run_vlm_parallel(pictures, ...):
       sem = Semaphore(4)
       async def bounded(pic, ix):
           async with sem:
               return await _describe_picture(pic, ix, ...)
       results = await asyncio.gather(*[bounded(p, i) for i, p in enumerate(filtered_pictures)], return_exceptions=True)
       return [r for r in results if isinstance(r, tuple)]

   image_texts = asyncio.run(_run_vlm_parallel(filtered_pics, ...))
   ```
4. Add exponential backoff (1s, 2s, 4s) on HTTP 429 responses inside `_describe_picture`.
5. Keep `vlm_called` counter; respect `vlm_max_images` before building `filtered_pics`.

### Task 3: Module-level DocumentConverter singleton
**File:** `chatSNP170226/backend/src/services/docling/orchestrator.py` (and/or `parser.py`)
**What:** Every Celery task invocation currently re-initializes `DocumentConverter` (loading weights, ~10–30s). Cache it at module level per worker process.
**How:**
1. At module level in `orchestrator.py`:
   ```python
   _CONVERTER_CACHE: dict[str, Any] = {}

   def _get_converter(pipeline_options) -> DocumentConverter:
       key = repr(pipeline_options)
       if key not in _CONVERTER_CACHE:
           logger.info("[docling] Initializing DocumentConverter singleton...")
           t0 = time.monotonic()
           _CONVERTER_CACHE[key] = DocumentConverter(pipeline_options=pipeline_options)
           logger.info(f"[docling] DocumentConverter ready in {(time.monotonic()-t0)*1000:.0f}ms")
       return _CONVERTER_CACHE[key]
   ```
2. Replace any `DocumentConverter(...)` instantiation inside `process_document_deep` with `_get_converter(...)`.
3. Add note in docstring: "Singleton persists for lifetime of worker process; `--max-tasks-per-child=50` ensures periodic refresh."

### Task 4: Per-stage elapsed_ms timing logs (structured JSON)
**File:** `chatSNP170226/backend/src/worker/media_tasks.py` + `chatSNP170226/backend/src/services/docling/orchestrator.py`
**What:** Emit a structured JSON log line at each major pipeline stage so docker logs / Prometheus scraper can aggregate timings.
**How:**
1. Define a helper in `helpers.py`:
   ```python
   import json, time, logging
   _perf_logger = logging.getLogger("chatsnp.perf")

   def log_stage(document_id: str, stage: str, elapsed_ms: float, extra: dict | None = None):
       _perf_logger.info(json.dumps({
           "event": "stage_timing",
           "document_id": document_id,
           "stage": stage,
           "elapsed_ms": round(elapsed_ms, 1),
           **(extra or {}),
       }))
   ```
2. In `media_tasks.process_document`, wrap each stage with `t0 = time.monotonic()` / `log_stage(doc_id, "parse", ...)` for stages:
   - `parse` — Docling `DocumentConverter.convert()`
   - `vlm` — entire VLM parallel block
   - `chunk` — `_smart_chunk()`
   - `embed` — mem0 embed calls
   - `upsert` — Qdrant upsert
   - `total` — full task duration
3. Include `chunks_count`, `page_count`, `file_size_kb` in `total` log.

## Threat Model / Risks
- **OpenRouter rate limit (429):** Capped at 4 concurrent + exponential backoff (1s→2s→4s) mitigates this. Worst case falls back to serial retry.
- **`asyncio.run()` inside Celery sync task:** Celery uses a thread pool by default; `asyncio.run()` creates a new event loop per call — safe but not reusing an existing loop. If worker is later changed to gevent/eventlet, must switch to `loop.run_until_complete`.
- **Singleton holds GPU/CPU memory:** DocumentConverter weights stay resident for `--max-tasks-per-child=50` tasks, then process is recycled. Acceptable tradeoff vs 10–30s cold-start per task.
- **`pipeline_options` key collision:** The `repr()` key may differ for equivalent options objects. Document that `_get_converter` should be called with a single canonical options instance.
