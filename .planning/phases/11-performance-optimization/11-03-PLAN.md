# Phase 11 Plan 03 — Progress SSE

## Goal

Currently users see a spinner with no feedback after uploading. A 2MB PDF can take 60–90s; without progress the UI feels broken. This plan adds a `progress` sub-object inside the existing `documents.meta` JSON column (no schema migration needed — `meta` is already a JSON dict), publishes stage events to Redis `document:{id}` channel from `media_tasks`, creates a new `GET /documents/{id}/stream` SSE endpoint, and wires a frontend progress bar with stage labels.

**Depends on:** 11-01 (per-stage `log_stage` helper and stage names must exist).

## Tasks

### Task 1: Define progress schema in `documents.meta` + update helper
**File:** `chatSNP170226/backend/src/worker/helpers.py`
**What:** No Alembic migration needed — `documents.meta` is already a JSON dict (line 103 in `models.py`). Define a standard progress sub-object shape and a helper to write it.
**How:**
1. Define the progress schema as a TypedDict for documentation:
   ```python
   # helpers.py
   STAGES = ["preflight", "parse", "vlm", "chunk", "embed", "upsert"]
   STAGE_WEIGHTS = {"preflight": 5, "parse": 25, "vlm": 20, "chunk": 10, "embed": 25, "upsert": 15}

   # meta.progress shape:
   # {
   #   "stage": "embed",          # current stage name
   #   "pct": 75,                 # 0-100 estimated percentage
   #   "elapsed_ms": 12400,       # ms since task start
   #   "chunks_so_far": 18,       # chunks embedded so far
   #   "updated_at": "2026-...",  # ISO timestamp
   # }
   ```
2. Add `update_document_progress(document_id: str, stage: str, elapsed_ms: float, chunks_so_far: int = 0)` to `helpers.py`:
   ```python
   def update_document_progress(document_id, stage, elapsed_ms, chunks_so_far=0):
       pct = sum(STAGE_WEIGHTS[s] for s in STAGES[:STAGES.index(stage)+1])
       progress = {
           "stage": stage,
           "pct": pct,
           "elapsed_ms": round(elapsed_ms, 1),
           "chunks_so_far": chunks_so_far,
           "updated_at": datetime.utcnow().isoformat(),
       }
       db_pool.execute_query(
           "UPDATE documents SET metadata = jsonb_set(COALESCE(metadata,'{}'), '{progress}', :val::jsonb) WHERE id = :id",
           {"val": json.dumps(progress), "id": document_id},
       )
       return progress
   ```

### Task 2: Publish progress events to Redis `document:{id}` channel
**File:** `chatSNP170226/backend/src/worker/media_tasks.py`
**What:** At each pipeline stage boundary in `process_document`, call `update_document_progress` and publish the result to Redis so SSE subscribers receive it in real time.
**How:**
1. Import redis client at top of `media_tasks.py`:
   ```python
   from src.core.redis_client import get_redis_client
   ```
2. Create a helper `_publish_progress(document_id, stage, elapsed_ms, chunks=0)`:
   ```python
   def _publish_progress(document_id, stage, elapsed_ms, chunks=0):
       from src.worker.helpers import update_document_progress
       progress = update_document_progress(document_id, stage, elapsed_ms, chunks)
       r = get_redis_client()
       r.publish(f"document:{document_id}", json.dumps({"type": "progress", **progress}))
   ```
3. Add calls in `process_document` after each stage completes:
   - After Docling `convert()`: `_publish_progress(document_id, "parse", elapsed_ms, page_count)`
   - After VLM block: `_publish_progress(document_id, "vlm", elapsed_ms)`
   - After `_smart_chunk()`: `_publish_progress(document_id, "chunk", elapsed_ms, len(chunks))`
   - During embed loop, every 5 chunks: `_publish_progress(document_id, "embed", elapsed_ms, chunks_embedded)`
   - After Qdrant upsert: `_publish_progress(document_id, "upsert", elapsed_ms, total_chunks)`
4. On task completion, publish `{"type": "done", "status": "ready", "total_chunks": N, "elapsed_ms": total}`.
5. On error, publish `{"type": "error", "message": "...Vietnamese error..."}`.

### Task 3: Add `GET /documents/{id}/stream` SSE endpoint
**File:** `chatSNP170226/backend/src/api/upload.py` (new route added here, or new `documents.py` API file)
**What:** SSE endpoint that subscribes to Redis `document:{id}` channel and forwards events to the client, following the same pattern as `GET /sessions/{id}/stream` in `chat.py`.
**How:**
1. Add route to `upload.py`:
   ```python
   from fastapi.responses import StreamingResponse
   from src.core.redis_client import get_redis_client
   import asyncio, json

   @router.get("/{document_id}/stream")
   async def stream_document_progress(document_id: str, user_id: str = Query(...)):
       """SSE stream for document upload progress."""
       async def event_generator():
           r = get_redis_client()
           pubsub = r.pubsub()
           pubsub.subscribe(f"document:{document_id}")
           try:
               while True:
                   msg = pubsub.get_message(ignore_subscribe_messages=True, timeout=1.0)
                   if msg and msg["type"] == "message":
                       data = msg["data"]
                       yield f"data: {data}\n\n"
                       parsed = json.loads(data)
                       if parsed.get("type") in ("done", "error"):
                           break
                   await asyncio.sleep(0.2)
           finally:
               pubsub.unsubscribe()
               pubsub.close()

       return StreamingResponse(event_generator(), media_type="text/event-stream",
           headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"})
   ```
2. Add auth check: verify `user_id` owns `document_id` via a quick DB query before subscribing.
3. Add 120s timeout: if no `done`/`error` event within 120s, yield `{"type":"timeout"}` and close.

### Task 4: Frontend — upload progress bar with stage labels
**File:** `chatSNP170226/frontend/src/` (upload component, likely `components/upload/` or `components/chat-sidebar.tsx`)
**What:** After receiving the upload response (document_id), open an SSE connection to `/documents/{id}/stream` and render a progress bar with stage label.
**How:**
1. Create custom hook `useDocumentProgress(documentId: string | null)`:
   ```typescript
   // hooks/useDocumentProgress.ts
   export function useDocumentProgress(documentId: string | null) {
     const [progress, setProgress] = useState<ProgressState | null>(null);
     useEffect(() => {
       if (!documentId) return;
       const userId = localStorage.getItem("user_id");
       const es = new EventSource(`/api/documents/${documentId}/stream?user_id=${userId}`);
       es.onmessage = (e) => {
         const data = JSON.parse(e.data);
         setProgress(data);
         if (data.type === "done" || data.type === "error") es.close();
       };
       return () => es.close();
     }, [documentId]);
     return progress;
   }
   ```
2. Add `UploadProgressBar` component:
   ```typescript
   // components/upload/UploadProgressBar.tsx
   const STAGE_LABELS: Record<string, string> = {
     preflight: "Kiểm tra dịch vụ...",
     parse:     "Phân tích tài liệu...",
     vlm:       "Mô tả hình ảnh...",
     chunk:     "Chia nhỏ nội dung...",
     embed:     "Tạo vector nhúng...",
     upsert:    "Lưu vào cơ sở tri thức...",
     done:      "Hoàn tất!",
   };
   ```
   Render: `<Progress value={progress.pct} />` + `<span>{STAGE_LABELS[progress.stage]}</span>`.
3. Wire `useDocumentProgress` into the existing upload flow: after upload POST returns `{document_id}`, pass it to the hook and show `<UploadProgressBar>` in place of the current spinner.
4. On `type === "done"`: hide bar, show success toast "Tải lên thành công! X chunks đã được xử lý."
5. On `type === "error"`: show error toast with message from event.

## Threat Model / Risks
- **Redis publish fails mid-task:** Progress publish is non-critical; wrap in `try/except` with `logger.warning` so a Redis blip doesn't fail the entire upload.
- **SSE connection limit:** Each open SSE tab holds a Redis pubsub subscription. Frontend hook must clean up on component unmount (`es.close()`). Add `maxConnections` note in component JSDoc.
- **`jsonb_set` SQL on non-JSONB column:** The `documents.metadata` column is `JSON` (not `JSONB`) per models.py. Use plain JSON update: `UPDATE documents SET metadata = :new_meta WHERE id = :id` where `new_meta = {**existing_meta, "progress": progress}`. Fetch existing meta first, merge in Python, then write back. Alternatively, run Alembic to change column type to `JSONB` (recommended for future).
- **VLM stage may be skipped:** When `DOCLING_VLM_ENABLED=false`, skip publishing `vlm` stage and adjust `STAGE_WEIGHTS` so percentages still sum to 100.
