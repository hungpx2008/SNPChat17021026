================================================================================

Latency Estimates (from user question to response):
  • Chat Mode: 2-5 seconds (direct LLM call)
  • RAG Mode: 5-15 seconds (search + synthesis)
  • SQL Mode: 5-20 seconds (agent loop + execution)

Throughput:
  • Concurrent chat sessions: ~50-100 (depends on hardware)
  • Concurrent file uploads: ~10 (1 worker, concurrency=1)
  • Task queue depth: Limited by Redis memory + Postgres write throughput

Memory Usage:
  • Embedding model: ~1.3 GB per Celery worker (loaded once)
  • Redis: ~100 MB baseline + session cache
  • PostgreSQL: ~500 MB baseline + data growth

Bottlenecks:
  1. Mem0 latency (300s timeout, can be slow for large fact bases)
  2. Docling processing (30-60s for large PDFs)
  3. LLM API latency (2-5s per call to OpenRouter)
  4. Qdrant search (fast, but filtered searches with many conditions can slow)

Scaling Strategy:
  • Horizontal: Add more Celery workers (all queues can scale)
  • Vertical: Increase hardware (CPU for embeddings, RAM for Mem0)
  • Caching: Redis already 1-hour TTL for sessions
  • Partitioning: Qdrant can be partitioned by department
